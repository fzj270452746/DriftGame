# Mahjong Drift
🧠 一、核心概念（Core Concept）

一句话定义：

一个“时间驱动的麻将数字解谜游戏”，所有牌的数值会随时间变化，玩家必须在“未来状态”中完成组合。

🎮 二、核心玩法循环（Core Loop）
进入关卡 → 随机生成麻将牌阵（带数字）
玩家执行操作：
选择牌
交换 / 连接 / 组合
回合结束 → 触发「数值漂移」
检测是否达成目标
重复直到：
✅ 完成目标（胜利）
❌ 步数耗尽或无解（失败）
🔢 三、基础规则（基础系统）
1. 牌定义

每张麻将牌包含：

花色（可选）
数值（1~9）

2. 数值漂移规则（核心机制）

每一回合结束时触发：

基础规则（标准模式）
所有牌：+1
超过9 → 回到1（循环）

👉 示例：

3 → 4
9 → 1
进阶规则（可作为关卡变化）
区域漂移：某些区域 +2
反向漂移：-1（1变9）
随机漂移：±1
延迟漂移：部分牌隔一回合才变化
3. 玩家操作方式（关键设计）

建议用以下三种之一（可混合）：

A. 交换（推荐）
每回合可交换2张牌位置
B. 路径连接
连线选中一条路径（类似连连看，但无消除）
C. 拖拽组合
把牌拖到“组合槽”

👉 推荐方案：
交换 + 自动检测组合（最清晰、最易上手）

4. 组合规则（核心解谜点）

可组合的规则（建议关卡逐步解锁）：

基础组合：
对子：相同数字 x2
顺子：连续数字（如 3-4-5）
刻子：相同数字 x3
数值目标型（推荐重点）
任意组合总和 = 指定数值（如 15）
连续组合总和 = 目标区间（如 10~20）
特殊组合：
环形（9→1→2）
对称组合（3-5-3）
奇偶组合
🎯 四、胜负条件设计
胜利条件（可组合）
完成指定组合数量
达成目标分数
在特定回合内完成
清除关键牌
失败条件
步数耗尽
时间耗尽（快节奏模式）
无法再形成有效组合（死局）
🧩 五、关卡设计体系（重点）
1. 关卡结构

每关定义：

初始牌阵（布局 + 数值）
漂移规则
操作限制（步数）
胜利条件
2. 关卡类型分类
类型1：精准解谜（Puzzle）
固定解
少步数
强策略

👉 类似残局

类型2：动态策略（Strategy）
多解
依赖漂移预判
类型3：生存模式（Survival）
无限生成
看能坚持多久
类型4：目标冲刺（Score Attack）
在有限回合拿高分
🧠 六、策略深度设计（让游戏“聪明”）
1. 预判系统（核心乐趣）

玩家需要思考：

“这一步不只是现在有效，而是2~3回合后会变成什么？”

2. 组合延迟收益
当前不成立
但下一回合成立

👉 形成“时间策略”

3. 风险决策
现在做小组合
还是等待大组合？
⚙️ 七、扩展机制（提升深度）
1. 特殊牌
🔒 锁定牌（不可移动）
❄️ 冻结牌（不漂移）
☣️ 污染牌（影响周围）
🎲 随机牌（每回合变数值）
2. 技能系统（适合App长期留存）

玩家可使用技能：

重置某张牌数值
冻结一回合
逆转漂移（-1）
指定组合直接完成
3. 连锁机制
多个组合连续触发：
奖励额外回合
或暂停漂移
🎨 八、UI & 交互建议（很关键）
1. 数值变化表现
数字跳动动画（+1）
颜色渐变（1→9）
2. 未来预览（强烈推荐）

👉 核心卖点！

长按牌：
显示“下一回合数值”
或开启：
未来1步预览模式
3. 时间感表现
回合结束时：
整体“脉冲动画”
强化漂移概念
📈 九、数值设计（平衡关键）
1. 数值循环的意义
9→1 形成“周期”
玩家必须：
利用周期，而不是对抗它
2. 难度控制手段
增加漂移速度
增加锁定牌
减少步数
提高目标复杂度
🚀 十、产品化方向（非常关键）
1. 关卡地图
线性推进 + 分支挑战
2. 每日挑战
固定种子 + 排行榜
